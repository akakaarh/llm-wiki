# 计算机体系结构基础

理解 CPU 流水线、总线架构、存储层次和 DMA，是理解嵌入式处理器工作原理的基础。这些概念贯穿从 Cortex-M 到 Cortex-A 的所有阶段。

---

## CPU 流水线（Pipeline）

### 基本概念

将指令执行分解为多个阶段，每个阶段由独立硬件处理，实现指令级并行。

### 经典五级流水线

```
时间 →  T1    T2    T3    T4    T5    T6    T7
指令1: 取指 → 译码 → 执行 → 访存 → 写回
指令2:       取指 → 译码 → 执行 → 访存 → 写回
指令3:             取指 → 译码 → 执行 → 访存 → 写回
```

| 阶段 | 英文 | 功能 |
|------|------|------|
| 取指 | IF (Instruction Fetch) | 从 Flash/SRAM 取指令 |
| 译码 | ID (Instruction Decode) | 解析操作码，读寄存器 |
| 执行 | EX (Execute) | ALU 运算、地址计算 |
| 访存 | MEM (Memory Access) | 读写数据内存 |
| 写回 | WB (Write Back) | 结果写回寄存器 |

### Cortex-M 的流水线

| 核心 | 流水线级数 | 说明 |
|------|-----------|------|
| Cortex-M0/M0+ | 2~3级 | 极简设计，低功耗 |
| Cortex-M3 | 3级 | 取指/译码/执行 |
| Cortex-M4 | 3级 | 同 M3，增加 FPU/DSP |
| Cortex-M7 | 6级 | 超标量，双发射 |
| Cortex-M55 | 4级 | 支持 Helium (MVE) |

### 流水线冒险（Hazard）

| 类型 | 原因 | 解决方案 |
|------|------|----------|
| 数据冒险 | 后续指令依赖前一条的结果 | 转发（Forwarding）、流水线停顿 |
| 控制冒险 | 分支指令改变 PC | 分支预测、延迟槽 |
| 结构冒险 | 硬件资源冲突 | 增加硬件资源 |

嵌入式中的体现：
- Flash 访问延迟可能导致流水线停顿（等待 Flash 准备数据）
- Cortex-M 的 ART（加速器）通过预取和缓存减少 Flash 延迟

## 总线架构

### AMBA 总线族

ARM 定义的片上总线标准，嵌入式中最常用：

| 总线 | 全称 | 特点 | 典型用途 |
|------|------|------|----------|
| AHB | Advanced High-performance Bus | 高速、流水线、突发传输 | CPU↔Flash、CPU↔SRAM、DMA |
| APB | Advanced Peripheral Bus | 低速、简单、无流水线 | UART、SPI、GPIO、定时器 |
| AXI | Advanced eXtensible Interface | 高性能、乱序、多通道 | Cortex-A、GPU、高速外设 |

### AHB 总线

```
         ┌──────────┐
         │   CPU    │
         └────┬─────┘
              │ AHB
    ┌─────────┼─────────┐
    │         │         │
┌───┴───┐ ┌──┴───┐ ┌───┴───┐
│ Flash │ │ SRAM │ │ AHB2APB│
│ 控制器│ │      │ │ Bridge │
└───────┘ └──────┘ └───┬───┘
                       │ APB
              ┌────────┼────────┐
              │        │        │
           ┌──┴──┐ ┌──┴──┐ ┌──┴──┐
           │UART │ │SPI  │ │GPIO │
           └─────┘ └─────┘ └─────┘
```

**AHB 特性：**
- 支持突发传输（Burst）：一次地址发送，多次数据传输
- 流水线操作：地址 phase 和数据 phase 重叠
- 多主机支持：通过仲裁器（Arbiter）管理总线竞争

**APB 特性：**
- 无流水线，每次传输需要至少两个周期（Setup + Access）
- 只支持单次传输，无突发
- 低功耗、低复杂度

### 地址映射

外设通过总线连接到 CPU 的地址空间：

```
0x0800_0000  Flash（程序存储）
0x2000_0000  SRAM（数据存储）
0x4000_0000  APB1 外设
0x4001_0000  APB2 外设
0x4002_0000  AHB1 外设（GPIO、DMA）
0xE000_E000  系统控制空间（NVIC、SysTick）
```

## 存储层次（Memory Hierarchy）

### 金字塔结构

```
速度 ↑    容量 ↓
┌─────────┐
│ 寄存器  │  ← 最快（1周期），最小（几十个32-bit）
├─────────┤
│  Cache  │  ← 快（几周期），较小（几KB~几MB）
├─────────┤
│  SRAM   │  ← 较快（1~2周期），中等（几KB~几MB）
├─────────┤
│  DRAM   │  ← 较慢（几十周期），大（几MB~几GB）
├─────────┤
│  Flash  │  ← 慢（读取有延迟），大（几MB~几GB）
└─────────┘
```

### 嵌入式存储器类型

| 类型 | 特点 | 典型用途 |
|------|------|----------|
| SRAM | 静态、无需刷新、快、贵 | 栈、堆、DMA 缓冲区 |
| DRAM | 动态、需刷新、慢、便宜 | 大容量数据存储（Cortex-A 系统） |
| Flash (NOR) | 按字节读取、可直接执行(XIP) | 程序代码存储 |
| Flash (NAND) | 按页读写、不能直接执行 | 大容量数据存储 |
| TCM | 紧耦合存储器、零等待周期 | 关键代码/数据（实时性要求高） |

### Cortex-M 的存储系统

```
CPU
 │
 ├── I-Code Bus ──→ Flash（取指）
 ├── D-Code Bus ──→ Flash（数据读取）
 └── System Bus ──→ SRAM、外设
```

- Cortex-M3/M4：Flash 有 ART 加速器，减少等待
- Cortex-M7：有 I-Cache 和 D-Cache
- TCM（Tightly Coupled Memory）：M7/M55 支持，用于关键代码

## 哈佛结构 vs 冯诺依曼结构

### 冯诺依曼结构

指令和数据共享同一总线和存储器。

```
CPU ←→ [总线] ←→ [统一存储器（指令+数据）]
```

- 优点：硬件简单，存储器利用率高
- 缺点：取指和取数据不能同时进行（总线瓶颈）

### 哈佛结构

指令和数据有独立的总线和存储器。

```
CPU ←→ [I-Bus] ←→ [指令存储器]
CPU ←→ [D-Bus] ←→ [数据存储器]
```

- 优点：取指和取数据可同时进行，吞吐量高
- 缺点：硬件复杂，成本高

### Cortex-M 的混合结构

Cortex-M 采用**改进的哈佛结构**：
- Flash 通过独立的 I-Code 和 D-Code 总线访问（哈佛特性）
- SRAM 通过统一的 System Bus 访问（冯诺依曼特性）

```
         ┌──────────┐
         │ Cortex-M │
         └┬───┬───┬─┘
          │   │   │
     I-Code  D-Code  System
     Bus     Bus     Bus
          │   │   │
     ┌────┴───┴───┴────┐
     │   Flash (XIP)    │ ← I-Code 和 D-Code 独立访问
     └──────────────────┘
          │
     ┌────┴──────────────┐
     │   SRAM            │ ← System Bus 访问
     └───────────────────┘
```

## DMA（Direct Memory Access）

### 原理

DMA 控制器可以在 CPU 不参与的情况下，在外设和内存之间（或内存与内存之间）搬运数据。

```
没有 DMA：
  CPU 读外设 → CPU 写内存 → CPU 读外设 → CPU 写内存 → ...
  （CPU 全程参与，无法做其他事）

有 DMA：
  CPU 配置 DMA → DMA 自动搬运 → DMA 完成中断通知 CPU
  （CPU 在搬运期间可执行其他任务）
```

### DMA 传输类型

| 类型 | 源 → 目标 | 应用 |
|------|-----------|------|
| 外设→内存 | 串口接收 → 缓冲区 | UART 接收 |
| 内存→外设 | 缓冲区 → 串口发送 | UART 发送 |
| 内存→内存 | 数组A → 数组B | 内存拷贝、初始化 |
| 外设→外设 | ADC → DAC | 信号处理链 |

### DMA 通道

DMA 控制器有多个通道（Channel），每个通道独立配置：

```c
// 配置 DMA 通道（简化示例）
DMA1_Channel5->CPAR  = (uint32_t)&USART1->DR;   // 外设地址
DMA1_Channel5->CMAR  = (uint32_t)rx_buffer;       // 内存地址
DMA1_Channel5->CNDTR = 64;                         // 传输数量
DMA1_Channel5->CCR   = DMA_CCR_EN | DMA_CCR_TCIE; // 使能 + 完成中断
```

### DMA 与 CPU 的协作

- **DMA 优先级**：DMA 和 CPU 可能同时访问同一总线，通过总线仲裁器决定优先级
- **内存屏障**：DMA 传输完成后，CPU 需要确保看到最新数据（`__DSB()`）
- **Cache 一致性**（Cortex-A）：DMA 操作物理内存，CPU 可能读到 Cache 中的旧数据，需要手动刷新

## 嵌入式系统的基本运行模型

```
┌─────────────────────────────────────────┐
│                CPU                       │
│  ┌─────────┐  ┌──────────┐  ┌────────┐ │
│  │ 流水线  │  │ 寄存器组 │  │ NVIC   │ │
│  │(取指译码│  │(R0-R15)  │  │(中断)  │ │
│  │ 执行)   │  │          │  │        │ │
│  └────┬────┘  └─────┬────┘  └───┬────┘ │
└───────┼─────────────┼───────────┼───────┘
        │             │           │
   ┌────┴─────────────┴───────────┴────┐
   │              总线矩阵              │
   │  (AHB/AXI，仲裁多主机访问)         │
   └──┬──────┬──────┬──────┬──────┬────┘
      │      │      │      │      │
   ┌──┴──┐┌──┴──┐┌──┴──┐┌──┴──┐┌──┴──┐
   │Flash││SRAM ││DMA  ││GPIO ││UART │
   │(XIP)││     ││     ││     ││     │
   └─────┘└─────┘└─────┘└─────┘└─────┘
```

1. CPU 从 Flash 取指（通过 I-Code Bus）
2. 译码后执行，可能读写 SRAM 或外设（通过 System Bus）
3. 外设事件触发中断，NVIC 跳转到 ISR
4. DMA 在后台搬运数据，完成后触发中断通知 CPU
5. 流水线让多条指令重叠执行，提高吞吐量
