# 数制与编码

嵌入式开发中，数据在底层以二进制形式存储和处理。理解数制转换和编码方案是阅读寄存器手册、分析内存布局的基础。

---

## 进制表示

### 常用进制

| 进制 | 基数 | 前缀（C语言） | 示例 |
|------|------|---------------|------|
| 二进制 | 2 | `0b` | `0b1010` = 10 |
| 八进制 | 8 | `0` | `012` = 10 |
| 十进制 | 10 | 无 | `10` |
| 十六进制 | 16 | `0x` | `0xA` = 10 |

### 进制转换

- **任意进制 → 十进制**：按权展开求和。`0x1A3 = 1×16² + 10×16¹ + 3×16⁰ = 419`
- **十进制 → 任意进制**：除基取余，逆序排列
- **二进制 ↔ 十六进制**：每4位二进制对应1位十六进制（`0b1101_0110` = `0xD6`）
- **二进制 ↔ 八进制**：每3位二进制对应1位八进制

嵌入式中最常用的是**十六进制**，因为寄存器地址、位域定义几乎都用十六进制表示。

## 整数编码

### 无符号整数（Unsigned）

n 位二进制表示范围：`0` ~ `2ⁿ - 1`

```
8-bit: 0 ~ 255
16-bit: 0 ~ 65535
32-bit: 0 ~ 4294967295
```

### 有符号整数 — 补码（Two's Complement）

计算机中负数用**补码**表示，好处是加法器无需区分正负号。

**补码规则（以8-bit为例）：**

| 表示 | 方法 | 8-bit 范围 |
|------|------|-----------|
| 正数 | 原码即补码 | `0` ~ `+127` (`0x00` ~ `0x7F`) |
| 负数 | 对应正数取反加1 | `-128` ~ `-1` (`0x80` ~ `0xFF`) |

**补码求法示例：**
```
+5  = 0b0000_0101
取反 = 0b1111_1010
加1  = 0b1111_1011 = -5 的补码 = 0xFB
```

**快速判断：** 最高位（MSB）为符号位，`0` 正 `1` 负。

**特殊值：**
- 8-bit: `-128` = `0x80`（无对应正数，取反加1后溢出）
- 补码零唯一：`+0` = `-0` = `0x00`

### 溢出（Overflow）

- 无符号溢出：最高位进位丢弃
- 有符号溢出：两个正数相加得负，或两个负数相加得正
- 嵌入式中需关注：计数器回绕、定时器溢出中断

## 浮点数 — IEEE 754

### 单精度（float，32-bit）

```
| 符号(1-bit) | 指数(8-bit) | 尾数(23-bit) |
|     S       |   E[7:0]    |  M[22:0]     |
```

- **符号位 S**：`0` 正 `1` 负
- **指数 E**：偏移量 127（实际指数 = E - 127）
- **尾数 M**：隐含前导1（1.M），精度约 7 位十进制

**示例：** `0x41200000` = `0 10000010 0100000...` = +10.0
- S=0, E=130, 实际指数=3, 尾数=1.01 = 1.25
- 值 = 1.25 × 2³ = 10.0

### 双精度（double，64-bit）

```
| 符号(1-bit) | 指数(11-bit) | 尾数(52-bit) |
```

精度约 15~16 位十进制。

### 特殊值

| 值 | 条件 | 含义 |
|----|------|------|
| ±0 | E=0, M=0 | 正零/负零 |
| ±∞ | E=0xFF, M=0 | 上溢 |
| NaN | E=0xFF, M≠0 | 非数（如 0/0） |
| 非规格化数 | E=0, M≠0 | 极小值，隐含前导0 |

### 嵌入式中的浮点

- Cortex-M4F/M7 有硬件 FPU（单精度），双精度需软件模拟
- 定点数（Fixed-Point）在无 FPU 的 MCU 上更常用：用整数模拟小数，如 Q16.16 格式

## 字符编码

### ASCII

7-bit 编码，共 128 个字符。

| 范围 | 内容 |
|------|------|
| 0x00~0x1F | 控制字符（`\n`=0x0A, `\r`=0x0D, `\t`=0x09） |
| 0x20~0x7E | 可打印字符（空格、数字、字母、标点） |
| 0x7F | DEL |

嵌入式常用：串口通信默认 ASCII，命令行协议（AT指令）用 ASCII。

### BCD（Binary-Coded Decimal）

用4位二进制编码1位十进制数。

- **压缩 BCD**：1字节存2位数字（`0x25` = 十进制25）
- **非压缩 BCD**：1字节存1位数字（高4位补0）

应用场景：RTC 时钟芯片（如 DS1307）的时间寄存器常用 BCD 格式。

### UTF-8

可变长度 Unicode 编码，兼容 ASCII。

| 码点范围 | 字节数 | 编码格式 |
|----------|--------|----------|
| U+0000~007F | 1 | `0xxxxxxx` |
| U+0080~07FF | 2 | `110xxxxx 10xxxxxx` |
| U+0800~FFFF | 3 | `1110xxxx 10xxxxxx 10xxxxxx` |
| U+10000~10FFFF | 4 | `11110xxx 10xxxxxx 10xxxxxx 10xxxxxx` |

嵌入式中用于：LCD 显示多语言文本、日志输出中文。

## 位操作速查

嵌入式编程中频繁使用位操作，此处列出核心操作：

| 操作 | C 表达式 | 说明 |
|------|----------|------|
| 置位 | `val \|= (1 << n)` | 第 n 位置 1 |
| 清位 | `val &= ~(1 << n)` | 第 n 位清 0 |
| 翻转 | `val ^= (1 << n)` | 第 n 位取反 |
| 读位 | `(val >> n) & 1` | 读取第 n 位值 |
| 掩码提取 | `val & 0xFF00` | 提取指定位段 |
| 位域赋值 | `(val & ~mask) \| (new << shift)` | 先清后写 |
